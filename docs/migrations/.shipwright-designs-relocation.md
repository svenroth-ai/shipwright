# `designs/` -> `.shipwright/designs/` Migration

Sub-Iterate F of the designs relocation flipped this artifact to
`status: migrated` in `shared/scripts/lib/artifact_migrations.py`. This
turns the SessionStart drift detector into a **hard gate** for any
project that still has a top-level `designs/` directory.

## What changed

- All Shipwright plugins now write design artifacts (mockups, manifest,
  visual guidelines, screen routes, chrome definition, design handoff)
  to `.shipwright/designs/` instead of `designs/` at the project root.
- Reads use only the canonical path -- there is **no Grace fallback**.
- The drift detector (`shared/scripts/hooks/check_artifact_drift.py`)
  scans for legacy `designs/` directories at SessionStart. With
  `status: migrated`, finding one exits non-zero with a structured JSON
  payload that AI orchestrators parse.

## What you need to do (existing projects with `designs/` at the root)

### Option 1 -- automated (recommended)

```bash
uv run shared/scripts/tools/migrate_artifact_dir.py \
  --artifact designs \
  --project-root /path/to/your/project
```

This:
1. Verifies `designs/` exists and `.shipwright/designs/` is empty (or absent).
2. Uses `git mv` if the legacy directory is git-tracked, falling back to
   `shutil.move` otherwise.
3. Refuses non-destructively if either precondition fails -- nothing moves
   until you resolve the conflict manually.
4. Pass `--dry-run` first to preview, then re-run without it to commit
   the move.

After the move, commit:

```bash
git add -A
git commit -m "refactor: relocate designs/ to .shipwright/designs/"
```

### Option 2 -- manual

```bash
# From the project root:
mkdir -p .shipwright
git mv designs .shipwright/designs  # if git-tracked
# OR
mv designs .shipwright/designs      # if untracked

git add -A
git commit -m "refactor: relocate designs/ to .shipwright/designs/"
```

## Wenn `designs/` Files in Git getrackt sind

`git status` zeigt nach dem Move noch die alten Files als deleted/added an.
`git mv` (Option 1 / 2 oben) handhabt das automatisch. Falls aber Files
manuell verschoben wurden:

```bash
git rm -r --cached designs/    # unstage alte Tracking
git add .shipwright/designs/   # neue Files stagen
git commit -m "refactor: relocate designs/ to .shipwright/designs/"
```

## What the drift detector does on a stale legacy folder

If a SessionStart hook fires in a project where `designs/` still exists
at the top level (and `status` for the migration is `migrated`):

1. The hook exits 1.
2. It writes structured JSON to stdout containing the finding plus a
   ready-to-paste `git mv` remediation list.
3. It also writes `.shipwright/stale-folders.md` with the same data so
   you see it in your editor.

The hook never touches your files -- it only reports.

## What if I see `.shipwright/stale-folders.md`?

That file is the human-readable summary of any drift findings. It's
auto-deleted as soon as the next SessionStart runs cleanly. If it
persists, follow the remediation steps inside the file (or in the JSON
the hook printed to stdout).

## Rollback

The migration is non-destructive (git history preserves both states).
To roll back to the legacy layout for a single project:

```bash
git mv .shipwright/designs designs
```

To roll back the framework-level decision (re-enable warn-only mode):
revert the status flip in `shared/scripts/lib/artifact_migrations.py`
back to `"in_progress"`. The drift detector will resume warn-only
behavior on the next SessionStart.
