1. **RTM layer columns** (`_rtm_layer_columns.py` + rtm_generator net-0). Per-FR
   `Unit | Integration | E2E` glyphs (`ok`/`MISSING`/`n/a`/`—`) appended AFTER `Status`
   (so `test_rtm_reconciled_column`'s positional `_cell(row, 7/8)` indices stay valid).

3. **`D-layer`** (`check_layer`). Active FR with a required layer lacking an
   executed-passing test (`coverage[layer] != "ok"`, R1). Provenance valve (carry-forward
   #2 / Spec §9): `explicit`⇒FAIL by priority (Must=HIGH/Should=MED/May=LOW) + `any_fail`;
   `legacy`⇒advisory (surfaced, `status="pass"`). `invalid_layers`⇒HARD. Fan-out: an `ok`

4. **D1 hardening** (`_check_d1` + `refine_d1_covered`). `covered` requires a *tested*
   event (`tests.total>0`) AND — for an `explicit`/post-rollout FR — a real non-ambiguous
   manifest test link. Two SEPARATE proofs keyed per-FR; legacy FRs keep the event-only
   proof so the pre-TT8 monorepo does not avalanche.

Bloat baseline NOT ratcheted. New/edited source ≤300; capped files net-neutral. Marketplace
cache re-synced (framework change).

### Notable design decisions (fail-closed reasoning — why no detector false-greens / false-reds)

- **D1 "split not joined" (binding §11).** Event-coverage and test-link-coverage are two
  SEPARATE sets, both required for an explicit FR, intersected per-FR — so an old tested
  event for FR-A + an unrelated tagged test for FR-B can never combine into a false
  `COVERED` (links are keyed to their OWN FR node; a fanned collision link is excluded).
  Can't false-green: a 0/0 event or a missing link drops the FR from `covered`. Can't
  false-red the monorepo: the link proof is provenance-gated to `explicit` FRs only (Spec
  §9 landmine — "hard for FRs created after the field ships"); all 15 monorepo FRs are
  legacy ⇒ event-proof only ⇒ D1 unchanged (empirically: still flags only FR-01.15).
- **D-layer requires executed-passing, not merely present (R1).** `coverage=="ok"` is a
  schema-structural proof of `enabled AND executed=pass`; a green-but-skipped test yields
  `MISSING`. Can't false-green a rotted (skipped) layer.
- **Namespace fan-out fail-closed (carry-forward #1).** A display id owned by ≥2 active
  requirements is ambiguous — one fanned tag credits both. D-layer treats such an `ok` as
  NOT confirmed (`ambiguous_fanout`); D1's link proof excludes collision ids; D-orphan
  cannot false-flag because the collector only orphans a tag resolving to NO live FR in
  ANY namespace. Pinned in both directions.
- **Manifest source (R3).** The detectives read the committed manifest (regenerated by
  `update_compliance` every phase); absent/malformed/non-v2 ⇒ SKIP (no proof ≠ pass), not
  a silent pass. The base+head-regenerating ENFORCING gates are TT5's F11 (this is a
  post-merge, non-blocking detective).
