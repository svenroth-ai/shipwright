# Iterate Spec: deterministic-render-timestamps

- **Run ID:** iterate-2026-05-22-deterministic-render-timestamps
- **Type:** bug
- **Complexity:** medium
- **Status:** draft

## Goal

Eight tracked markdown files (`.shipwright/agent_docs/build_dashboard.md`,
`session_handoff.md`, `triage_inbox.md`, all 5 `.shipwright/compliance/*.md`)
are re-rendered with a `Generated:`/`Updated:`/`Auto-generated` header
containing `datetime.now(UTC)`. Even when the underlying data is
unchanged, every Stop hook / iterate F5b call mutates the wall-clock
banner, leaving the file `M` in `git status`. The Shipwright dev-loop
permanently sees 8-9 dirty tracked files; `git pull` blocks; operator
workflow degrades. Replace the wall-clock value with a deterministic
function of the events log — two renders against the same
`shipwright_events.jsonl` produce byte-identical output.

## Acceptance Criteria

- [ ] AC-1: A new `latest_event_dt(project_root) -> datetime | None`
      helper lives in `shared/scripts/lib/events_log.py`. Returns the
      max `ts` field across all events in the worktree-resolved
      `shipwright_events.jsonl`, parsed as a UTC `datetime`. Returns
      `None` when the log is missing or empty. Tolerant of corrupt
      lines (skip them, like `_read_event_log` already does).
- [ ] AC-2: Three `shared/scripts/tools/*` renderers consume the helper
      and use ISO timestamp from it for their banner instead of
      `datetime.now()`:
      - `update_build_dashboard.py::generate_dashboard` (both legacy +
        event-based code paths, two call sites)
      - `aggregate_triage.py::main` (fallback when `--now` CLI arg is
        not provided)
      - `generate_session_handoff.py::generate_handoff_content`
      Each preserves its current strftime display format.
- [ ] AC-3: The compliance plugin's `data_collector.collect_all()`
      stops calling `datetime.now(...)` for `ComplianceData.timestamp`.
      It reads events.jsonl via the existing `_read_event_log`
      (already worktree-aware) and computes `max(e["ts"] for e in events)`
      locally. Falls back to `"(no events)"` literal when the log is
      empty, matching what the 5 compliance generators render in
      their `Generated:` line. The compliance copy stays separate
      from the shared helper per the existing parity pattern (the
      compliance plugin is a distinct distributable; cross-plugin
      `shared/scripts/lib` import would need a path bootstrap).
- [ ] AC-4: A new parametrized test
      `shared/tests/test_render_determinism.py` proves:
      - For each of 4 callers (`update_build_dashboard`,
        `aggregate_triage`, `generate_session_handoff`, and the
        compliance generators wrapper): rendering twice against the
        same synthetic `tmp_path/shipwright_events.jsonl` yields
        byte-identical output (`run_1 == run_2`).
      - Different inputs → different outputs (sanity check: adding
        one event to the fixture between the two runs produces
        different output, proving the determinism isn't just because
        the helper returns a constant).
- [ ] AC-5: An empirical probe before F0 demonstrates that running
      `finalize_iterate.py` twice in the same worktree session
      produces zero diff in
      `{.shipwright/agent_docs/build_dashboard.md, .shipwright/compliance/*.md,
      .shipwright/agent_docs/session_handoff.md}` between the two runs
      (modulo a sleep between calls to prove it's not just same-now-clock
      luck). Empirical, end-to-end, against real renderers — the
      Confidence-Calibration anti-pattern guard.

## Spec Impact

- **Classification:** NONE
- **NONE justification:** Behaviour-preserving rendering refactor.
  The semantic content of every report is unchanged — only the

## Affected Boundaries

`shipwright_events.jsonl` ist die einzige beteiligte serialisierte Datei.
Diese Iterate LIEST sie nur (Helper holt max(ts)). Kein Producer-Wechsel.

| Producer (writes) | Consumer (reads) | Format |
|---|---|---|
| F7 `record_event.py` (unchanged) | `events_log.latest_event_dt` (NEW) | JSONL line-per-event with `ts` ISO8601 field |
| F7 `record_event.py` (unchanged) | `data_collector._read_event_log` → `collect_all` (modified) | same |

Pattern: `latest_event_dt` parst nur das `ts`-Feld pro Zeile; `json.loads`
+ `event.get("ts", "")`. Korrupt-Zeilen werden silent geskippt (wie
existierender `_read_event_log` pattern). ISO8601 lexicographic-sort
ist chronologically-equivalent (für `Z`/`+00:00`-suffix beide), also
`max(ts)` ergibt korrekt den jüngsten Event.

## Confidence Calibration

- **Boundaries touched:** Read-only against `shipwright_events.jsonl`.
  Producer (F7 `record_event.py`) is unchanged. Two consumers added:
  `events_log.latest_event_dt` (shared helper) + the compliance
  plugin's local `_latest_event_timestamp` mirror.
- **Empirical probes run:**
  1. **Helper unit-test matrix (11 cases in `test_events_log.py`):**
     missing log → None ✓; empty log → None ✓; single event → its ts ✓;
     multiple → max ✓; `Z` vs `+00:00` suffix mix → correct ✓; corrupt
     middle line → skipped, max from valid ✓; no `ts` field → skipped ✓;
     non-string `ts` → skipped ✓; unparseable `ts` string → skipped ✓;
     naive datetime → treated as UTC ✓; all-corrupt → None ✓.
  2. **Cross-rendering determinism (`test_render_determinism.py`):** 6
     parametrized cases (3 renderers × 2 directions) — same input →
     identical output (`time.sleep(1.2)` between runs), different
     input → different output (sanity).
  3. **Compliance-side determinism (`test_data_collector_determinism.py`):**
     16 parametrized cases — 4 cover `_latest_event_timestamp` directly,
     5 cover each generator with same-input-byte-identical, 5 cover
     different-input-different-output, plus 2 cover `collect_all`
     timestamp source.
  4. **End-to-end empirical probe (the real one):** ran
     `finalize_iterate.py` twice in this worktree, separated by
     `time.sleep(2)`, then diffed all 7 markdown files
     (build_dashboard + session_handoff + 5 compliance/*.md). Result:
     **7 of 7 byte-identical**. The remaining 9th dirty file is
     `shipwright_events.jsonl` (the F7-strand, out of scope).
  5. **Parity probe** (compliance vs shared helper): the
     `_latest_event_timestamp` unit-test class in
     `test_data_collector_determinism.py::TestLatestEventTimestamp`
     mirrors the shared helper's 4 base cases. Both helpers return
     the same string for the same input.
- **Edge cases NOT probed + why acceptable:**
  - **Concurrent write to events.jsonl during the read:** the helper
    reads line-by-line and catches `JSONDecodeError` per-line, so a
    half-written final line is skipped silently. The events log is
    append-only with locking on the producer side
    (`shipwright_events.jsonl.lock`), so a torn write only happens on
    the *current* tail line. The helper's worst case is "the very
    latest event is missed for one render", which is acceptable for a
    banner.
  - **Future-dated events:** if a clock-skewed event has `ts` in the
    future, `max()` would surface it. Acceptable — the helper returns
    the chronologically-latest *known* event, including clock-skewed.
    Audit trail still has the raw value.
  - **Sub-second precision drift:** events use second-precision in
    practice (`record_event.py` strftime); no fractional-second
    rounding hazards.
- **Confidence-pattern check:** Single "are you confident?"-shaped
  prompt fired from user before this iterate ("alles empirisch
  verifiziert?"). That answered "no" → one more probe added (the
  end-to-end finalize-twice empirical probe), which finished with
  zero diff across all 7 files. Asymptote-heuristic satisfied — most
  recent probe returned no finding AND all applicable categories
  covered AND no yes-then-bug pattern fired this run.
