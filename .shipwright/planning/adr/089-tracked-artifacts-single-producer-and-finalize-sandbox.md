# ADR 089 — Runtime/snapshot split for agent-doc trio + finalize sandbox gate

## Status
Accepted — 2026-05-27

## Context

Three tracked Markdown files under `.shipwright/agent_docs/` —
`session_handoff.md`, `build_dashboard.md`, `triage_inbox.md` — were
re-generated by every Stop hook in every plugin (12 plugins × per-turn
frequency). Verified empirically in session `40b1eb76` end-state: 8 dirty
files in main, 49 Stop hooks fired in the last turn. Every long-lived
branch produced merge conflicts on these paths, and `git status` was
never clean on main.

PR #78 (iterate-2026-05-23-compliance-md-single-producer) closed the
analogous regression class for the 5 compliance MDs by making
iterate-finalize the sole producer and deleting Stop-hook fallback
writes. That delete-only fix worked for compliance because those docs
are archival in nature — staleness between iterates is benign. But the
agent-doc trio is *live mid-session state*: non-iterate sessions
(security work, manual fixes, design work) consume fresh handoff +
triage between iterates. A delete-only fix would have lost that value.

A second escape was found during verification of this iterate's design:
`iterate_stop_finalize.py:_active_worktree_root()` returns `None` when
the session→worktree pointer is stale or missing. The code then fell
through to `resolve_project_root()` which uses cwd — which at Stop-time
is the main repo. Finalize wrote the 8 tracked compliance + agent-doc
MDs into main, bypassing PR #78's single-producer guarantee at the
target level.

A third operational concern was rebase: `audit_staleness.find_snapshot_commit`
locates the last single-producer snapshot via `git log --grep=Run-ID:`.
Rebase rewrites Run-ID commit SHAs and (depending on strategy) drops
merge-bearing trailers entirely. The audit then reports
`snapshot_unavailable` on a branch with dozens of legitimate Run-ID
commits.

## Decision

**Runtime/snapshot split.** Extend the PR #78 single-producer pattern
with a runtime-vs-tracked split:

- Stop hooks write live state to `.shipwright/agent_docs/runtime/`
  (gitignored via re-exclude in the `.gitignore` whitelist block).
- Iterate-finalize is the sole producer of the tracked variants.
- Per-file decision in finalize avoids the "always generate then
  overwrite" double-producer anti-pattern (external review OpenAI #1):
  - `session_handoff.md` and `build_dashboard.md` — finalize ALWAYS
    direct-writes the tracked path (they carry iterate-specific context:
    canon marker, run_id, that the runtime Stop-hook variant lacks).
    The runtime variant is then unlinked.
  - `triage_inbox.md` — finalize copies runtime → tracked via atomic
    `os.replace`, then unlinks runtime. If runtime is absent (fresh
    worktree, first iterate after onboarding), finalize calls
    `aggregate_triage.main()` to seed tracked directly.

**Hard-gated repair pass.** `iterate_stop_finalize.py` step 4 (the
finalize repair pass) now refuses to run when `_active_worktree_root()`
returned None — no fallback to cwd. The pointer validation is
strengthened: resolved path must exist, live under `main_repo_root`,
and carry a `.git` file/dir (rejects foreign repos / stale pointers
to deleted worktrees / absolute paths to other clones).

**Audit coverage extension.** `audit_staleness.DOC_REGISTRY` widens
from 5 to 8 docs (the 3 agent-doc paths join). `find_snapshot_commit`
queries both `.shipwright/compliance/` AND `.shipwright/agent_docs/`
path filters. A new `not-in-snapshot` semantic handles the case where
a doc was added to the registry AFTER the most recent snapshot commit:
both on-disk and snapshot side missing → benign; on-disk exists +
snapshot missing → also benign (the next finalize will introduce the
path, subsequent audits compare normally).

**Branch integration convention.** `docs/hooks-and-pipeline.md` gains
a "Branch integration" section codifying `git merge` (not `git rebase`)
for Run-ID-bearing branches, with operational guidance and recovery via
`git reflog`. Drift-protected by `shared/tests/test_branch_integration_doc.py`.
Doc-only enforcement; programmatic pre-rebase guard deferred per external
review OpenAI #12.

**Path-safety helpers.** `shared/scripts/lib/artifact_paths.py` exposes
the runtime/tracked constants (`AGENT_DOCS_DIRNAME`, `RUNTIME_SUBDIR`,
`TRACKED_AGENT_DOC_NAMES`), the `runtime_path` / `tracked_path` helpers,
and `ensure_path_within_project_root` which constrains `--out-dir`-style
flags against path-escape and symlink-redirect attacks. `aggregate_triage.py`
now refuses an `--out-dir` that escapes the project root or has a symlink
component (TOCTOU mitigation).

**Atomic write + tempfile cleanup.** `finalize_iterate._atomic_replace`
writes via `tempfile.NamedTemporaryFile` + `os.replace`. On failure the
tempfile is unlinked so it doesn't leak as visible `.session_handoff.md
.…tmp` cruft under the tracked agent_docs/ dir (the gitignore allowlist
doesn't exclude them).

## Consequences

- Eliminates the "main is dirty after every session" regression class.
  Stop hooks no longer touch tracked agent_docs/.
- Mid-session inspection is preserved: operators can `cat .shipwright/
  agent_docs/runtime/<file>.md` for fresh state; tracked carries the
  last iterate's snapshot.
- The repair pass refuses to write the main tree when the session
  pointer is invalid. Stale-pointer sessions no longer poison main.
- Group E (`audit_staleness`) covers the agent-doc trio identically
  to compliance MDs. False-positive class from "registry expansion
  ahead of snapshot" is closed via `not-in-snapshot` semantic.
- Future iterates must use `git merge` (not `git rebase`) when
  integrating main into Run-ID branches. `gh pr merge --rebase` carries
  the same risk as a local rebase.
- 17 new tests + 8 updated tests + 1 plugin pyproject normalisation.
  Iterate plugin gained `[project.optional-dependencies] dev = [pytest...]`
  to match the convention used by the other 10 plugins.

## Rationale

Alternative A — delete Stop-hook writes entirely (1:1 PR #78 pattern) —
was rejected because session_handoff + triage_inbox are live mid-session
state, not archival snapshots. Deleting Stop writes would have:

1. Lost mid-session inspect value for non-iterate sessions (security,
   manual fixes, design work). These are CURRENT consumers of
   handoff/triage state between iterates.
2. Broken the `iterate_stop_finalize.py:_dashboard_reflects_run_id`
   freshness-gate consumer that reads `build_dashboard.md` at Stop-time.
3. Stranded the `build_dashboard.md` consumer pattern documented in
   ADR-088 (the dashboard is consulted by operators between iterates).

Alternative B — pre-commit hook stripping non-Run-ID commits' agent_docs
changes — was rejected because (a) target-project end-users would inherit
it and be surprised, (b) it papers over the underlying produce-by-Stop
bug rather than fixing root cause, (c) it competes with the existing
bloat anti-ratchet pre-commit hook for the `core.hooksPath` slot.

The per-file decision in finalize (snapshot for triage, direct-write
for handoff/dashboard) avoids the "always generate then overwrite" anti-
pattern: when both paths produce the same tracked file, byte-divergence
becomes possible if either path's generation logic changes without the
other being updated. The asymmetric per-file path is intentional and
documented in `_unlink_runtime_artifacts` + `_snapshot_triage_runtime`.

## Rejected alternatives

- **Alternative A: delete Stop-hook writes entirely.** See Rationale.
- **Alternative B: pre-commit hook with Run-ID gate.** See Rationale.
- **Alternative C: programmatic pre-rebase guard.** Deferred per external
  review OpenAI #12. The doc-only convention may prove sufficient; if
  the audit_staleness false-positive class recurs from rebased branches,
  a future iterate can add a pre-rebase hook keyed on Run-ID trailer
  presence.
