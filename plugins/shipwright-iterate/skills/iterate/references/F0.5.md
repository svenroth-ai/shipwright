# F0.5 ‚Äî End-to-End Verification Gate

> **Authoritative anchor.** The F0.5 section in SKILL.md keeps the
> `Mandatory at medium+` declaration, the four fail-closed conditions
> (`tests_run`, `exit_code`, `justification`, `surface`), references to
> both `surface_verification.py` (orchestrator) and
> `verify_iterate_finalization.py` (post-commit audit), and the
> `Backend-affects-Frontend` rule ‚Äî all of which are pinned by
> `tests/test_skill_e2e_gate_consistency.py`. This file carries the
> expanded prose only.

This is the single authoritative gate verifying that the user-erlebbare
surface was empirically driven through a running stack. Steps 9 and 11
produce early signal; F0.5 is what F6 commits against. A non-zero exit
from F0.5 is **STOP** ‚Äî do not proceed to F1.

## Step 1 ‚Äî Determine Behavior Surface

Pick exactly one (or `none` with justification):

- `web` ‚Äî Playwright against `dev_server.py` (default for web profiles)
- `cli` ‚Äî scripted CLI / skill / pytest invocation against fixture
- `api` ‚Äî HTTP probe against running server (API-only changes, no UI)
- `none` ‚Äî no startable surface; `justification` is mandatory and the
  reason is referenced in the iterate ADR

## Step 2 ‚Äî Run the Surface Runner

See `references/design-and-testing.md` ‚Üí "End-to-End Verification ‚Äî
Execution" for the per-surface protocol. Inherits the 3-retry
browser-fixer pattern from build's Step 4.5; cap at 3, fail-closed
after.

```bash
uv run "{shared_root}/scripts/surface_verification.py" \
  --project-root "{project_root}" \
  --run-id "{run_id}" \
  --surface "{web|cli|api|none}" \
  [--justification "..."]   # required when surface=none
```

The orchestrator writes raw evidence to
`{project_root}/.shipwright/runs/{run_id}/surface_verification.json`.

## Step 3 ‚Äî Stage Evidence for F5

F5 consolidates the F0.5 raw output into
`shipwright_test_results.json.iterate_latest.surface_verification`:

```json
"iterate_latest": {
  ...,
  "e2e": { ... },
  "surface_verification": {
    "surface": "web|cli|api|none",
    "runner": "<command that ran>",
    "exit_code": 0,
    "tests_run": 0,
    "evidence_path": "<path to log/screenshot/playwright-report>",
    "timestamp": "<ISO8601>",
    "justification": "<required when surface=none>"
  }
}
```

Backwards-compat: existing readers (compliance `data_collector.py`,
`test_evidence.py`) only see the new key when they read it ‚Äî old
readers do not break.

## Fail-closed conditions

`surface_verification.py` exits non-zero on any of:

1. `surface != "none"` AND `tests_run == 0` (greedy filter matched zero
   specs ‚Äî critical Playwright failure mode where `--grep` mismatch
   still returns exit 0).
2. `exit_code != 0` from the runner after the 3-retry cap.
3. `surface == "none"` without a `justification`.
4. The `surface_verification` block is missing entirely at medium+ when
   no `surface: none` opt-out is recorded (post-commit audit in
   `verify_iterate_finalization.py`).

A non-zero exit at F0.5 means STOP ‚Äî do **not** proceed to F1. F0.5 is
the production-time chokepoint; the post-commit audit in
`verify_iterate_finalization.py` is the second layer.

## Backend-affects-Frontend rule

If the diff touches API routes, store mutations, SSE / WS handlers,
message contracts, or any code consumed by the UI ‚Äî `surface = web` is
mandatory even when no `client/**` file changed. The `always` cell in
the Phase Matrix subsumes detection: at medium+, the gate runs
regardless of file paths.

## Spec-only authorship is regression-equivalent to no test

Authoring a Playwright / pytest / curl spec without executing it
counts as `tests_run = 0` and fails the gate. The "always" semantics
in the matrix mean **author AND run**, not author OR run.

## Silent-skip re-check (Python + TS/JS)

A `tests_run > 0` surface can still hide a rotted layer if the change
`test.skip`-ed or `.only`-focused a spec. So the test-hygiene probe is
**re-run at F0.5** over the changed test files ‚Äî not advisory-only ‚Äî as the
same handle the Self-Review ¬ß 8 uses:

```bash
uv run shared/scripts/tools/scan_test_hygiene.py --diff
```

A non-zero exit is **STOP** (same as any F0.5 failure). It covers Python
silent-skip-without-CI-guard AND changed TS/JS specs (Playwright / Vitest /
Jest): a `test.skip` / `it.skip` / `describe.skip` / `test.fixme` / `test.todo`
/ `xit` (incl. chained `.only.each` / `.skip.each` / `.concurrent.only`) must
carry a `@quarantine` annotation (`reason` + `owner` + `ticket` +
`expires: YYYY-MM-DD`; an expired or >180-day-out `expires` fails), and a
focused test (`.only` / `fit` / `fdescribe`) is an **unconditional** failure ‚Äî
never quarantineable. A runtime conditional `test.skip(cond, 'reason')` (first
arg not a string) is exempt. The check is diff-scoped by line, so a
pre-existing untouched skip does not block. This CLI (via ¬ß8 / F0.5) is the
single mechanical gate ‚Äî the F11 verifier does not re-run it.

**Coverage limit:** the TS/JS leg detects specs by name (`*.spec.*` /
`*.test.*` / `*.e2e.*`) or an `e2e/` / `test(s)/` / `__tests__/` dir. A project
with a fully custom `testMatch` / `include` outside those, or a skip reached
via computed / bracket-notation / aliased access, is out of the limited-syntax
scope (the repo-wide skip inventory is a TT7/TT8 concern).
