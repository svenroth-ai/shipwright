# F11 — Push Branch, Open PR & Verify

**Leak-guard first** — confirm the run never touched the main repo
working tree (snapshot-and-diff against the B1a Step-1 snapshot):

```bash
uv run "{shared_root}/scripts/checks/check_iterate_isolation.py" \
  --project-root "{project_root}" --run-id "{run_id}" --stage f11
```

Non-zero exit = **STOP**.

**Push the iterate branch and open a PR.** An iterate run NEVER checks
out, merges, or pushes the default branch — that races every other
parallel iterate. One iterate = one branch = one PR.

```bash
default_branch=$(git -C "{project_root}" symbolic-ref refs/remotes/origin/HEAD 2>/dev/null | sed 's@^refs/remotes/origin/@@' || echo main)
git -C "{project_root}" push -u origin "iterate/{slug}"
# gh has no `-C` analog — it acts on the cwd's repo. cd into the worktree
# so gh can never operate on the main repo even if the shell cwd drifted.
cd "{project_root}"
gh pr create --base "$default_branch" --head "iterate/{slug}" \
  --title "<type>(<scope>): <description>" \
  --body "<summary>

## Empirical Test Completeness
<derived from iterate_latest.test_completeness — e.g.>
Testable: N · Tested: N · Untestable (structural reason): M · Untested-but-testable: 0 ✅

Run-ID: {run_id}"

# --- Arm GitHub-native auto-merge (B4.5 Phase 3) -----------------------
# An iterate PR squash-merges itself once the Required Checks pass. The
# arm is gated TWO ways so it can never run away:
#   (1) branch scope — ONLY `iterate/*` arms. A manual, human-authored
#       PR never reaches this prose, and the `case` guard means it would
#       not self-arm even if it did (opt in per-PR with your own
#       `gh pr merge --auto` or an `automerge` label instead).
#   (2) fail-soft — if "Allow auto-merge" / branch protection is OFF in
#       repo settings (or the branch can't merge), `gh pr merge --auto`
#       errors. We WARN and leave the PR open. NEVER `|| exit`: a missing
#       repo setting must not break F11 for every future iterate.
head_branch="iterate/{slug}"
pr_url=$(gh pr view "$head_branch" --json url -q .url)
case "$head_branch" in
  iterate/*)
    gh pr merge "$pr_url" --auto --squash --delete-branch \
      || echo "WARN: auto-merge not armed (repo 'Allow auto-merge' off / no branch protection / conflict) — PR left open for manual merge: $pr_url" ;;
  *)
    echo "Non-iterate branch '$head_branch' — auto-merge NOT armed (manual opt-in only)." ;;
esac
```

**Empirical Test Completeness line (always, small+).** Render one line from
`iterate_latest.test_completeness` into the PR body so the reviewer reads the
answer to *"did you test everything testable?"* without asking. `Untested-but-testable`
is `0` by construction — the F11 verifier (below) fails closed otherwise.
At trivial complexity emit `Completeness: n/a — <justification>`.

If `iterate/{slug}` has diverged from `origin/<default>` since B1a
fetched it (another iterate merged meanwhile), rebase onto current
`origin/<default>` and resolve conflicts BEFORE `gh pr create` — that
rebase is the deliberate, single conflict point. **This run STOPS here:
it never merges the PR itself.** The PR is **armed for auto-merge** (the
block above) and GitHub squash-merges it automatically once the Required
Checks go green. If the arm can't be set (repo "Allow auto-merge" off,
branch protection absent, or a merge conflict) the PR stays open for a
manual, human-gated merge — the iterate is finished either way.

**Gate check:** F5b records the `work_completed` event into **this
worktree's own** `shipwright_events.jsonl` (the per-tree, PR-committed model —
`events_log.resolve_events_path` returns the literal `{project_root}` copy),
and F6 stages it so it ships in the PR. The deterministic verifier below checks
this authoritatively via `check_events_has_commit`, which — when the log is
tracked — asserts the event is in the **committed HEAD blob**, not merely the
working copy (AC4). So the event MUST be at `{project_root}/shipwright_events.jsonl`
AND committed by F6; if F6 forgot to `git add` it, the verifier fails closed.

**Deterministic verifier.** After the gate check, run the
finalization verifier and fail the iterate run on red:

```bash
uv run "{shared_root}/scripts/tools/verify_iterate_finalization.py" \
  --run-id "{run_id}" \
  --project-root "{project_root}" \
  --commit "$(git -C "{project_root}" rev-parse HEAD)"
```

Exit 0 = green (or warnings only), exit 1 = at least one required
artifact missing. Add `--strict` to treat warnings as errors.

This run includes `check_test_completeness_ledger` (small+): it fails closed
when any behavior in `iterate_latest.test_completeness` is testable-but-untested,
an `untestable` row lacks a closed-vocabulary `reason_code`, or the
enumeration falls short of the AC count. That is the gate that makes the
operator's pre-merge "did you test enough?" question unnecessary.
