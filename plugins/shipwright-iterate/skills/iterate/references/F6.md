# F6 — Commit (Conventional Commits)

- **Feature:** `feat({scope}): {description}`
- **Change:** `refactor({scope}): {description}` or `feat` if user-facing
- **Bug:** `fix({scope}): {description}`

By this point F3, F3a, F4, F5, F5a, F5b, and F5c have all written
their artifacts into the working tree. Stage everything explicitly
rather than using `git add -A` (avoids accidentally picking up
unrelated dirty files):

```bash
# Code + tests (run-specific paths)
git add <source files edited in build>
git add <test files added/modified in build>

# Finalization artifacts (always)
git add {project_root}/CHANGELOG-unreleased.d/       # F4 drop files (one or more)
# NOTE: do NOT stage F3 decision-drops here. `write_decision_drop.py` writes them
# next to the MAIN repo (gitignored, worktree-aware via resolve_main_repo_root),
# never into the worktree — so `git add .shipwright/agent_docs/decision-drops/`
# exits 128 in a worktree iterate (and is a gitignored no-op on main). They are
# consumed by `aggregate_decisions` at `/shipwright-changelog` release time.
git add {project_root}/.shipwright/agent_docs/build_dashboard.md
git add {project_root}/.shipwright/agent_docs/iterates/          # F5c entry + any migration quarantine
git add {project_root}/shipwright_test_results.json
git add {project_root}/shipwright_run_config.json
git add {project_root}/shipwright_events.jsonl       # F5b work_completed event — per-tree, MUST ship in the PR (skip only if gitignored in this project)
git add {project_root}/.shipwright/triage.jsonl      # tracked triage backlog — producers append during the iterate; deltas ship in the PR (skip only if gitignored in this project)

# Conditional finalization artifacts
git add {project_root}/.shipwright/agent_docs/conventions.md       # if F3a wrote learnings
git add {project_root}/.shipwright/agent_docs/architecture.md      # if F2 flagged structural impact
git add {project_root}/.shipwright/planning/**/spec.md             # if F1 flagged drift
git add {project_root}/.shipwright/planning/iterate/*.md           # if medium+ (iterate spec / mini-plan)
git add {project_root}/.shipwright/planning/iterate/<run_id>/      # review record (reviews.json) — the *.md glob above does NOT match it
git add {project_root}/.shipwright/planning/iterate/campaigns/<slug>/status.json  # if a campaign sub-iterate (F5b Step 6 re-projected the per-tree board — campaign S3)

# Compliance artifacts (if the project tracks .shipwright/compliance/)
git add {project_root}/.shipwright/compliance/

git commit -m "<type>(<scope>): <description>

<body>

Run-ID: {run_id}
Co-Authored-By: Claude <noreply@anthropic.com>"
```

If a path in the list above doesn't exist in the current project, skip
that `git add` — it just means that particular artifact wasn't touched
by this run. Never add `-A` to bypass the list.

> **`shipwright_events.jsonl` is mandatory when tracked.** Since
> iterate-2026-05-29-events-jsonl-worktree-commit the event log is a per-tree
> artifact: F5b records the `work_completed` event into **this worktree's**
> copy, and staging it here is what carries the event into the PR (and to
> `main` on merge). Skipping it strands the event outside the PR — exactly the
> bug that fix closed. The F11 verifier (`check_events_has_commit`) fails if a
> tracked log's event is not in the commit. Only skip the `git add` when the
> project **gitignores** `shipwright_events.jsonl`.

> **`.shipwright/planning/iterate/<run_id>/reviews.json` is the same**
> (iterate-2026-07-21-review-record): the review record is written into the
> worktree during the run, and the `*.md` glob above does NOT match it. A record
> left unstaged is deleted with the worktree after the PR merges, so the Mission
> view renders an empty Review row — the precise "nobody wrote it down" outcome
> the record exists to abolish. The F11 verifier (`check_review_record`) fails
> when the record is tracked but absent from the commit, mirroring
> `check_events_has_commit`. The run-scoped `external_*review_state.json`
> markers alongside it ARE gitignored by design and correctly stay out.

> **`.shipwright/triage.jsonl` is the same** (tracked since campaign
> 2026-06-05-track-triage-jsonl): pipeline producers append findings to the
> worktree copy during the iterate; staging it ships those deltas in the PR.
> Concurrent appends across worktrees reconcile via `resolve_churn_conflicts`
> (`_reconcile_triage`: exact-line dedup + header/JSON validation), so no manual
> conflict resolution is needed. Background Stop-hook / on-demand `triage_add`
> writes to the **main** repo's durable backlog are leak-guard-exempt (they are
> durable-log writes, not a branch leak). Skip the `git add` only if the project
> gitignores `triage.jsonl`.

> **Campaign `status.json` is the same per-tree pattern** (campaign
> 2026-06-07-tracked-campaign-status, S3): for a campaign sub-iterate, F5b Step 6
> re-projects `.shipwright/planning/iterate/campaigns/<slug>/status.json` from
> this worktree's event log and staging it ships the producer-owned board in the
> PR (it was previously a write-once main-tree file that never reached a fresh
> clone / the deployed WebUI). Concurrent-sibling conflicts reconcile via
> `resolve_churn_conflicts` (glob-admitted, placeholder side then
> regenerate-from-events). Stage it only when this run is a campaign sub-iterate
> (the `campaign`/`sub_iterate_id` stamp in F5b).
