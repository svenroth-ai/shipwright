# F2 — Architecture / Convention Update (conditional)

Check ANY of:

- New route / endpoint, OR existing route gaining a materially new behavior
- New component / page
- New schema / migration
- New service / subprocess
- **New write surface** (any new file path the project writes to, even one file
  under an existing dir)
- **New read surface** (any new external file or API the project reads from)
- New convention (naming, config layout, file location)

If yes: add ONE compact bullet to the right always-loaded doc AND pass the
matching `--architecture-impact` flag to `write_decision_drop.py` in F3.

## Where it goes (routing — ONE home per impact)

Routing is the single source of truth `lib.architecture_doc.IMPACT_TARGETS`,
shared by the producer (`write_decision_log.py`), this skill, and the F11 gate +
Group-F detective — so they cannot diverge:

| `--architecture-impact` | Doc | Section |
|---|---|---|
| `component` | `.shipwright/agent_docs/architecture.md` | `## Architecture Updates` |
| `data-flow` | `.shipwright/agent_docs/architecture.md` | `## Architecture Updates` |
| `convention` | `.shipwright/agent_docs/conventions.md` | `## Convention Updates` |

Put a `convention` entry under **`## Convention Updates` in `conventions.md`** —
NOT in architecture.md. (`## Architecture Updates` is for `component` /
`data-flow` only.)

## How it reads — compact, one line, point don't paste (progressive disclosure)

These docs are **always-loaded Layer-1 context**: every line costs tokens on
every future iterate. The detail lives ONCE in the on-demand ADR (the F3
decision-drop, ≤500 chars/field) and, for anything longer, the
`.shipwright/planning/adr/<NNN>-<slug>.md` spec folder. The agent-doc bullet is a
**pointer**, not a copy. One line:

```
- **<run_id>** (YYYY-MM-DD): <Impact> — <one sentence: what changed + key
  surface/module>. → decision_log.md (ADR via Run-ID: <run_id>)
```

- **Budget:** ≤ 600 chars, one logical line — same discipline `write_decision_log.py`
  enforces on `decision_log.md`. The rule is enforced **repo-agnostically** (this
  monorepo AND every adopted repo): the F11 verifier `check_agent_doc_budget`
  STOPs finalize on an over-budget NEW/changed entry, and the standalone CLI
  `shared/scripts/tools/check_agent_doc_budget.py` (SSoT `lib.agent_doc_budget`,
  closes the run-id-slug date hole) gives you the same check by hand. **After
  writing THIS entry (and the F3a `## Learnings` line) run the check locally
  before push** — it sits outside the `shared/tests` F0 run, so otherwise an
  over-budget entry surfaces only as a RED CI on the PR (the 2026-06-12 #213
  "shoot-and-forget" miss — memory `feedback_no_shoot_and_forget`):
  ```bash
  uv run shared/scripts/tools/check_agent_doc_budget.py --project-root .   # forward-only vs git base, any repo
  uv run --extra dev pytest plugins/shipwright-iterate/tests/test_agent_doc_entry_rules.py  # monorepo full-corpus
  uv run --extra dev pytest shared/tests/test_artifact_path_canon.py  # F0-stage, but F0 runs before F2 writes — re-run
  ```
- **Don't restate the ADR.** File paths, test names, rejected alternatives, and
  rationale belong in the decision-drop / spec file. If the bullet is growing
  past one sentence, move the prose into `--spec-ref` and keep the bullet to
  "what + pointer".
- The `run_id` (or the eventual `ADR-NNN`) is the anchor the gate matches on —
  always include it verbatim.
