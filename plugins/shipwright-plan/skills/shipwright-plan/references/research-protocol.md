# Research Protocol

## Purpose

Before planning, understand the technical landscape. Research informs all subsequent steps.

## Codebase Research (existing project)

1. **Structure scan**: Read directory tree, identify key patterns
2. **Key files**: Read entry points, config files, package.json/pyproject.toml
3. **Patterns**: Identify conventions (naming, folder structure, testing approach)
4. **Dependencies**: Review installed packages and their versions

## New Project Research

1. **Technology stack**: Review docs for chosen frameworks
2. **Best practices**: Search for patterns in similar projects
3. **Constraints**: Check compatibility between chosen technologies

## Web Research

Use web search for:
- Unfamiliar APIs or libraries
- Migration guides between versions
- Known issues or gotchas with specific technology combinations

## Output

No file written. Research forms the mental model for planning.
Capture key findings in interview notes if they influence design decisions.
