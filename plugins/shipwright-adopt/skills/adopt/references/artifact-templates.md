# Artifact Templates — Slot Mapping

Adopt uses the same templates as `/shipwright-project` for structural
consistency. No divergence in shape or headings.

## CLAUDE.md

Source: `shared/templates/claude-md-template.md` (today filled via
`artifact_writer._render_claude_md`, not the template string — same
structural output).

| Slot | Source |
|---|---|
| `{PROJECT_NAME}` | Project root dir name |
| `{FRONTEND_STACK}` etc. | `snapshot.stack.{frontend,backend,database,auth,runtime}` keys (comma-joined) |
| `## WHAT` body | `enrichment.product_description` (Layer-2) OR fallback prose |
| `## HOW` commands | `snapshot.commands.{dev,build,test}` |
| `## Ongoing Changes` note | Static block pointing to `/shipwright-iterate` |
| Profile pointer | `snapshot.profile.matched` |

## agent_docs/architecture.md

Source: `shared/templates/agent-docs/architecture.md.template` (slot
names: `{ARCHITECTURE_DIAGRAM}`, `{DATA_FLOW_DESCRIPTION}`).

| Slot | Source |
|---|---|
| Stack table | `snapshot.stack.*` |
| `{ARCHITECTURE_DIAGRAM}` | ASCII box drawing from Layer-2 (inline), or deterministic fallback via `_render_ascii_diagram_fallback` |
| `{DATA_FLOW_DESCRIPTION}` | `enrichment.architecture_prose` |
| Layers Detected section | `snapshot.folders.layers[]` |
| Profile pointer | `snapshot.profile.matched` |

**Style**: ASCII in fenced code block — match `webui/agent_docs/architecture.md`.
No Mermaid in V1 (V2 follow-up epic rolls out Mermaid + arc42 together).

## e2e/flows/adopted-baseline.spec.ts

Generated by `e2e_baseline_generator.py` when `routes.json` exists.
One test per crawled route. Each test checks:
- Route loads (HTTP 200)
- `h1` matches what was seen during crawl
- Top-3 buttons/CTAs visible
- (Does NOT assert form submission, auth flow, etc. — would be hallucination)
