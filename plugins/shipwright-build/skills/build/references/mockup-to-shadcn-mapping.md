# Mockup → shadcn/ui Component Mapping

Translation table from HTML mockup patterns (generated by `/shipwright-design`) to shadcn/ui implementation code.

Use this when implementing UI sections: find the mockup HTML pattern, apply the corresponding shadcn/ui composition.

---

## Mapping Table

| Mockup HTML Pattern | shadcn/ui Implementation | Reference Block |
|---|---|---|
| `<div class="stat-card">` with title + value + trend | `Card` > `CardHeader` > `CardDescription` + `CardTitle` (tabular-nums) + `CardAction` > `Badge variant="outline"` | dashboard-01/section-cards |
| `<table class="data-table">` with toolbar + filter + pagination | DataTable (tanstack/react-table) + `Table` + `TableHeader` + `TableBody` + `DropdownMenu` for row actions | dashboard-01/data-table |
| `<div class="card-grid">` with repeating cards | `grid grid-cols-1 gap-4 sm:grid-cols-2 lg:grid-cols-3` with `Card` per item | — |
| `<nav class="sidebar">` with groups + links | `SidebarProvider` + `Sidebar` + `SidebarMenu` + `SidebarMenuItem` + `SidebarMenuButton` | sidebar-07 |
| `<header>` with logo + user avatar | Site header with `Avatar` + `DropdownMenu` for user menu | dashboard-01/site-header |
| `<nav class="tabs">` with content panels | `Tabs` + `TabsList` + `TabsTrigger` + `TabsContent` | — |
| `<form>` with labeled inputs | `Card` > `CardContent` > `FieldGroup` > `Field` + `FieldLabel` + `Input`/`Select`/`Textarea` | preview/shipping-address |
| `<form>` with side-by-side fields | `FieldGroup className="grid grid-cols-2"` nesting | preview/shipping-address |
| `<div class="settings-row">` with label + toggle | `Field orientation="horizontal"` + `Switch` or `Checkbox` + `FieldContent` > `FieldLabel` + `FieldDescription` | preview-02/notification-settings |
| `<div class="empty-state">` with icon + text + button | `Empty` > `EmptyHeader` > `EmptyMedia` + `EmptyTitle` + `EmptyDescription` + `EmptyContent` > `Button` | preview/no-team-members |
| `<div class="modal">` or `<div class="dialog">` | `Dialog` + `DialogContent` + `DialogHeader` + `DialogTitle` + `DialogDescription` | — |
| `<div class="sheet">` or side panel | `Sheet` + `SheetContent` + `SheetHeader` + `SheetTitle` | — |
| `<span class="badge badge-success">` | `Badge variant="outline"` or `Badge variant="secondary"` (never raw color classes) | — |
| `<div class="alert">` or callout | `Alert` + `AlertTitle` + `AlertDescription` | — |
| `<div class="chart-container">` | `Card` > `ChartContainer config={chartConfig}` > Recharts component | preview/analytics-card |
| `<div class="progress-bar">` | `Progress value={percent}` | — |
| `<div class="skeleton">` or loading placeholder | `Skeleton className="h-4 w-3/4"` | — |
| `<div class="avatar">` with image | `Avatar` + `AvatarImage` + `AvatarFallback` (always include fallback) | — |
| `<select>` dropdown | `Select` + `SelectTrigger` + `SelectContent` + `SelectGroup` + `SelectItem` | — |
| `<input type="checkbox">` in a group | `FieldSet` + `FieldLegend` + `Field orientation="horizontal"` + `Checkbox` | — |
| Buttons with icons | `Button` + icon with `data-icon="inline-start"` (no sizing classes on icon) | — |

## Layout Mapping

| Mockup CSS Pattern | Tailwind Implementation |
|---|---|
| `display: grid; grid-template-columns: repeat(3, 1fr)` | `grid grid-cols-3 gap-4` |
| `display: grid; grid-template-columns: repeat(auto-fill, minmax(300px, 1fr))` | `grid grid-cols-1 gap-4 sm:grid-cols-2 lg:grid-cols-3` |
| `max-width: 1200px; margin: 0 auto; padding: 0 24px` | `mx-auto max-w-7xl px-6` |
| `display: flex; flex-direction: column; gap: 24px; padding: 24px` | `flex flex-col gap-6 p-6` |
| Sidebar + main content | `SidebarProvider` + `Sidebar` + `SidebarInset` |
| Centered card (auth page) | `flex min-h-svh items-center justify-center p-4` |

## Fallback Rule

If no mapping exists for a mockup pattern:
1. Infer the closest shadcn/ui component from the Component Selection table in `shadcn-rules.md`
2. Follow the composition rules (full Card structure, Items in Groups, etc.)
3. Add a code comment explaining the inference: `// Mapped from mockup: [description of pattern]`
