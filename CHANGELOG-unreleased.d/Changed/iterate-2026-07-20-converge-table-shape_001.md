`/shipwright-project` and `/shipwright-adopt` now emit one identical functional-requirements table: `| ID | Area | Name | Priority | Description | Basis | Layers |`. `Area` is a display label derived from the requirement's own ID group, so it can never disagree with it, and `Basis` replaces the old `Source` file path with a short controlled answer to "how do we know this?" — `interview`, `code`, `observed`, `tests`, `assumed` or `other`. `assumed` is the honest option when nobody has confirmed a requirement, so a guess stops reading as established fact.
