**`.shipwright_toolcall_count` renamed to `.shipwright/toolcall_count`** (commit `283f604`, 2026-04-29). Drive-by cleanup of the one outlier flagged in the just-shipped configs-migration deferral memo: every other root-level Shipwright artefact follows `shipwright_*_config.json` or `shipwright_events.jsonl`, while `.shipwright_toolcall_count` was a plain-integer telemetry counter with no extension and no `shipwright_` prefix. Renamed to `.shipwright/toolcall_count` (no leading dot in the basename — redundant inside an already-hidden dir). Scope was 28 hits in 10 files (4 code: `track_tool_calls.py` writer, `estimate_context_pressure.py` reader incl. argparse default, `reset_tool_counter.py` resetter, `orchestrator.py:586` between-skill cleanup; 3 tests; `plugins/shipwright-build/skills/build/SKILL.md` user-facing `--counter-file` flag in two places; `.gitignore`; this changelog reference). All write sites now call `parent.mkdir(parents=True, exist_ok=True)` before `write_text` so first-time-after-upgrade projects don't crash on missing `.shipwright/` dir. The legacy `.shipwright_toolcall_count` entry stays in `.gitignore` under a "legacy path - do not remove" comment block (separate-line per Lesson 6 of the artifact-migration pattern). **User-impact for existing Shipwright projects:** on first invocation after upgrade the counter resets to 0 in the new path; the old root-level file becomes dead weight, gitignored, harmless. The `.shipwright/<file>` path is automatically picked up by the existing `.gitignore` `.shipwright/` rule — no new ignore entry needed. The wider configs/events.jsonl body (~582 hits / ~132 files) remains deferred per the companion memo.
