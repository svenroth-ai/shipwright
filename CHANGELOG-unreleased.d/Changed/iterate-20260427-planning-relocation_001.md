**Planning artifacts relocated from `planning/` to `.shipwright/planning/`** -- a single hidden project-state directory at the project root, parity with the prior `securityreports/` -> `.shipwright/securityreports/` move. All Shipwright plugins (project, plan, design, build, iterate, adopt, compliance, run, test) read and write planning artifacts under the canonical path; no Grace-Period fallback. New SessionStart hook `shared/scripts/hooks/check_artifact_drift.py` scans for stale legacy `planning/` directories and hard-gates (exit 1 + structured JSON to stdout + `.shipwright/stale-folders.md` report) when found in projects whose migration status is `migrated`. Existing projects can run `uv run shared/scripts/tools/migrate_artifact_dir.py --artifact planning [--dry-run]` to relocate cleanly (uses `git mv` when tracked, refuses non-destructively if canonical destination has content). User-facing migration doc: `docs/migrations/.shipwright-relocation.md`. Generic pattern reference for future artifact migrations: `docs/migrations/artifact-migration-reference.md`.
