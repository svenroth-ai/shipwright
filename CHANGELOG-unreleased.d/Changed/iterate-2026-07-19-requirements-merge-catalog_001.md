Requirements are now stated once, in one catalog, in plain business language: the per-change `Refined by <run>` prose is gone (417 lines down to 286 for the same fifteen requirements), and requirement text no longer carries a run identifier, a decision-record number or a file path. Nothing is lost — which changes touched a requirement is answered from the event log, and the reasoning stays in the planning document that produced it. Each requirement now emits an explicit `<a id="fr-0101"></a>` anchor, so the traceability matrix's deep links resolve; previously all fifteen scrolled nowhere, because the viewer matches anchors exactly and a heading slug does not equal the fragment the matrix emits. Duplicate-requirement-ID detection (Group I check I4) is now global rather than per-document — a project whose several requirement documents legally reuse ID numbers will newly fail; see `docs/migrations/requirements-catalog-merge.md`.
