The `security:` job from the Supabase/Next.js CI template. It ran `AikidoSec/github-actions-wf@v1`, an action whose repository no longer exists (404), so the job could never have run for anyone adopting this template — and it additionally demanded an `AIKIDO_SECRET_KEY` secret nobody holds. Security scanning for adopted repos lives in `security.yml`, which `/shipwright-adopt` scaffolds independently, so this removes a false claim of coverage rather than coverage.
