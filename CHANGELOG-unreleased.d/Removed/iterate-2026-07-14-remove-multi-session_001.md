Multi-session pipeline mode and its external per-phase-session engine. `/shipwright-run` used to hand each phase off to its own `claude --session-id` session, driven by three hooks (`phase_session_start`, `phase_user_prompt_validate`, `phase_session_stop`) wired into all 8 phase plugins — so the pipeline could not advance in the VS Code extension or desktop chat, which cannot spawn a bound session. Those hooks (and their helpers `phase_context_blocks`, `hook_session`, `phase_event_emit`) are deleted and deregistered.
