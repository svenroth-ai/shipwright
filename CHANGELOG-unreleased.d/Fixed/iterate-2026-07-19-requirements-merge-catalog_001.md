Two compliance-audit checks stopped reporting a clean result about a set they never examined. The layer-coverage detective (D-layer) used to report "every active FR is covered at its required layers" when the traceability manifest declared no requirements at all — indistinguishable, to a reader, from a genuinely fully-covered project; it now skips with a detail naming the state, which does not affect the audit's exit code. The stale-reference detective (D2) used to skip when a project had no readable requirements, which is exactly the state in which every requirement reference in the change log is broken; it now reports those references. **D2's change can newly fail an audit:** a project with no readable requirements AND at least one recorded change naming a requirement now exits non-zero, and the dashboard reads "FAIL — drift found". D2 still skips when there are no references to check, so a project that simply has not written requirements yet is unaffected. Where requirements exist but their table could not be read, the report now names the table-shape defect instead of accusing the operator of stale references.
