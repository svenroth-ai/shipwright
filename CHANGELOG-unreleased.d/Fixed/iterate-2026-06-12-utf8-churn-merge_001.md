Churn-merge reconciliation no longer corrupts `.shipwright/triage.jsonl` on Windows: `resolve_churn_conflicts` now decodes git output as strict UTF-8 so a hard triage conflict unions byte-identically (was a UTF-8 -> cp1252 -> UTF-8 round-trip writing mojibake into the tracked log), and a genuinely-corrupt legacy byte now fails loudly with a structured `triage_invalid` status + merge abort instead of a traceback. `integrate_main` wraps its merge and follow-up `git commit` calls so a pre-commit-hook rejection returns a structured JSON status (`merge_commit_failed` / `followup_commit_failed`) and aborts cleanly instead of crashing with no JSON and a wedged `MERGE_HEAD`.
