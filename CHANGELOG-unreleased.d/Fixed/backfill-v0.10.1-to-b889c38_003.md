**Adopt crawler page-isolation + smart API mock + locator timeouts.** The single-page BFS pattern silently cascade-failed all routes after one "Target page closed" — every subsequent route attempted on the dead page also failed. The blanket `200 []` API-mock crashed object-shape endpoints (`/api/health`, `/api/diagnostics`) on `data.something`. Each route now runs in a fresh `context.newPage()` (closed in `finally`); GET / HEAD / OPTIONS pass through to the real backend (consumers see real response shapes), writes are stubbed with `{}` 200 to keep the crawl side-effect-free; locator-timeouts tightened to fail-fast on selector-mismatch instead of consuming the 180 s test budget (commit `6315929`).
