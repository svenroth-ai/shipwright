The event log no longer loses records to a damaged line. When two events ended up on the same physical line — which an ordinary git union merge can cause when one side's file lacks a trailing newline — the reader skipped the whole line and BOTH events disappeared. Because that log is the audit trail, a lost work_completed made a step that really happened read as one that never did. Records on such a line are now recovered in order, and the event writers check for an unterminated line before appending. Extends the same fix already shipped for the triage log.
