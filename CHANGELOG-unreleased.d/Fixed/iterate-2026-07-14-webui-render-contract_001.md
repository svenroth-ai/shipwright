**`/shipwright-adopt`: `git.major_refactor_commits` was always empty, for every repository.** `git log --numstat` prints a blank line between a commit's header and its file stats; the parser treated that blank as end-of-commit and reset its state before a single stat line was read, so the "5+ files changed" threshold was unreachable. Refactor commits are now detected, and the header fields are separated by `0x1F` so a `|` in a commit subject no longer shifts the recorded date and author.
