**adopt route-crawler Windows path bug**: `shared/scripts/route_crawler.py` produced spec paths with backslashes (`e2e\_shipwright-adopt-crawler.spec.ts`) on Windows, which `npx playwright test` interprets as a regex with escape sequences and fails to match. Now converts to forward slashes before passing to the subprocess. Regression test `shared/tests/test_route_crawler_windows_path.py` runs in the default fast suite to prevent future drift. Found by manual real-Playwright slow test during the adopt-fix campaign's verification phase.
