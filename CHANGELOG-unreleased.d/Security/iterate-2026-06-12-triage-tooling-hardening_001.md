Terminal-escape injection into the triage surfaces is closed: the triage_inbox.md renderer and `triage_cli list` now strip control characters from item title/detail/evidence (previously only launchPayload was sanitized), so an attacker-influenceable GitHub workflow name or branch can no longer inject ANSI/OSC escape sequences that execute when an operator views the inbox in a TTY (F31). The shared `_strip_control_chars` helper now also strips the C1 control range (0x80-0x9F, including the single-byte CSI 0x9B) in addition to C0/DEL, while preserving non-control Unicode (>= 0xA0).
