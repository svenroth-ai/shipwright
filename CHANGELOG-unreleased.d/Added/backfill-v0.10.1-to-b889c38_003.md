**Universal Rollback Discipline as first-class Deploy-Profile concept (Phase 0 / I2 / T5).** Extracts the three universal rollback patterns (revertable, provenance, procedure) from the Jelastic-specific deploy implementation into Deploy Profiles — a sibling concept to Stack Profiles. Ships: `shared/profiles/deploy-profile.schema.json` (JSON-Schema Draft 2020-12 encoding revertable/provenance/procedure as schema-required mechanism fields, with a `confidence: verified | documented | inferred` discipline marker); three reference profiles (`jelastic.json` shipped + verified, `vercel.json` stub + documented, `compose-vps.json` stub + documented); pure validator library (`shared/scripts/lib/deploy_profile_validator.py`) plus CLI (`shared/scripts/tools/validate_deploy_profile.py`) with structural + cross-field semantic checks; new "Rollback-Discipline (Universal)" section in deploy `SKILL.md` + companion `references/rollback-discipline.md` mapping pattern × target. The Jelastic implementation itself is unchanged — this iterate is discipline + schema, not new live deploy capability. External dual-LLM plan-review (Gemini + OpenAI) gated the architecture decision to "Option B": discipline booleans dropped in favor of mechanism fields + confidence value rules enforced at validator level (commit `c68eea0`).
