**Performance budget gate (Phase 0 / I3 / T3)** — new `performance` test category in `/shipwright-test` (Step 3.8). Lighthouse via Playwright-Chromium (no extra browser install — reuses the project's existing E2E Chromium) plus a pure-Python gzipped bundle-size measurement on `build_output_dir`. Defaults: Lighthouse Performance score ≥ 85, LCP ≤ 2500ms, bundle ≤ 250KB gz, gate `warn` (logs failures, pipeline continues; opt into `block` for hard failure). Override per-project via `shipwright_test_config.json.performance` with deep-merge precedence (`lighthouse.min_score` overrides without losing inherited `lcp_max_ms`). Five documented skip modes (profile-opts-out, no dev_url, no build artifacts, no bundle match, lighthouse_unavailable) — never silent-pass. Lighthouse pinned to `13.1.0` with committed lockfile. Sandbox stays ON by default; `SHIPWRIGHT_PERF_NO_SANDBOX=1` opt-in for CI/Docker. Test seam (`SHIPWRIGHT_PERF_LHCI_FAKE`) gated by `SHIPWRIGHT_TEST_MODE=1` + fake-fixture loadability check; misconfiguration WARNs and falls through to real Lighthouse (commit `05dc2c0`).
