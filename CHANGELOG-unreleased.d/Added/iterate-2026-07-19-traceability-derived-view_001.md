Ask which changes touched a requirement, and get an answer you can check. `uv run shared/scripts/tools/fr_history.py FR-01.11` lists the recorded changes for a requirement, oldest first, from the project's change log — replacing the hand-maintained history that used to sit inside the requirement text and go stale there. A requirement nothing has touched says "no recorded changes" rather than showing an empty list, and an ID that names no requirement is reported as unknown instead of looking like a requirement nothing ever touched. A requirement that has since been retired can still be asked about. Every answer states how much of the change log carries requirement links at all, and points at the commit history for what it cannot account for, so a short answer reads as thin records rather than as a stable capability.
