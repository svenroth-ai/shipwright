`accepted_risks_cli.py converge` resolves `github-dismissal` register entries against live GitHub code scanning, so one recorded acceptance quiets the security tab as well as triage instead of leaving the repo red for a risk somebody consciously accepted. Alerts are matched on `(tool, rule, path)` — never the rule alone, which was measured unsafe on this repo — and each dismissal carries the register's own justification plus a `[shipwright-accepted-risk: <id>]` marker. Read-only unless `--apply`; deliberately not wired into CI, because no scheduled job may hold the authority to mass-dismiss security alerts.
