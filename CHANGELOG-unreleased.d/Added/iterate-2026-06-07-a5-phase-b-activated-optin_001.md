Group A5.6 (CI security-workflow audit) gains an `a5_phase_b_activated` opt-in read from `audit_config.json`: when `true`, a project that has deliberately activated Phase B (active `pull_request:` / `schedule:` triggers) PASSES instead of emitting the permanent LOW "non-dormant trigger active" false-positive. The two structural guards — `workflow_dispatch:` presence and the bare-`on:`/no-triggers failure — still fire regardless of the flag, so the opt-in is narrower than `disabled_checks:[A5.6]`. Default `false` keeps dormant-by-default for freshly `/shipwright-adopt`-ed repos (convention-lock default `A5_PHASE_B_ACTIVATED_DEFAULT` in `shared/scripts/lib/security_workflow.py`).
